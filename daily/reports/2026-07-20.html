大模型与 Agent

Cloudflare 发布「Agent Native Cloud」愿景(The Agent-Native Cloud)S1

把 Agent 时代的云基础件重新定义为四原语:沙盒(执行)、会话(状态)、记忆(持久)、网关(流量)。Workers 全家桶对应改造,AI Gateway 新增跨 350+ 模型的统一计费与缓存。

点评:这不是营销话术,是定价权之战——谁定义 Agent 的「操作系统层」,谁收租。记忆原语直接对标你在做的方向,值得逐句读原文。

记忆被抬到云原语高度,等于给你的 MemoryNexus 方向盖章;可重点拆它会话/记忆的 API 边界设计,直接借鉴。

来源:Cloudflare Blog 7/18 · HN 热榜

DeepSeek 两周消耗 3508 亿 token,内部定调 All in AgentS1

晚点披露:DeepSeek 内部 token 消耗量两周达 3508 亿,约 7 成流向 Agent 类任务;梁文锋在内部会上明确「下一阶段 All in Agent」,V4 重点强化长程任务与工具调用。

点评:3508 亿这个数字比任何 PR 都有说服力——Agent 负载已经在吃真实算力,不是 PPT 需求。推理侧的多租户调度会成为所有厂的生死题。

3FS/smallpond 是 DeepSeek 为 Agent 负载备的数据底座,你的 Lakehouse 视角正好能解释它为什么此时加码存储层。

来源:晚点LatePost 7/19

OpenAI 重组基础设施团队,押注多租户推理S1

The Information:OpenAI 将 Infra 团队重组为「推理平台」与「训练平台」两条线,推理线核心 KPI 是多租户并发下的单 token 成本,目标年内再降 40%。
